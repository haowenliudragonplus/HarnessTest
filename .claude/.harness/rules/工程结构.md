## 主工程（不热更，慎改）

`Assets/Scripts/Main/` 是 **MonoBehaviour 入口层**，不会被 HybridCLR 热更，修改它需重新打包 App。

### 关键文件
- [Boot.cs](../../../Assets/Scripts/Main/Boot.cs) — 启动入口，挂载 Procedure 流水线，桥接 MonoBehaviour 生命周期到热更工程
- `Const_Boot.cs` — 主工程常量
- `Loader.cs`（推断存在）— 桥接静态委托：`OnUpdate` / `OnFixedUpdate` / `OnLateUpdate` / `OnSecondsUpdate` / `OnHalfSecondsUpdate` / `OnApplicatePause` / `OnApplicateQuit`

### 修改原则
- 仅在以下情况修改主工程：
  1. 新增 MonoBehaviour 生命周期事件需要桥接到热更
  2. 新增 SDK / Plugin 必须在主工程初始化
  3. HybridCLR 启动流程调整
- 任何主工程改动**都不能热更**，必须通过 App 重新发版生效——风险高

## 热更工程（业务核心）

`Assets/Scripts/Hotfix/` 是所有业务代码所在，会被编译为 DLL 通过 HybridCLR 热更。分为四个子层：

### Framework 层（基础设施）
`Assets/Scripts/Hotfix/Framework/Module/`

提供横切能力的"基础设施 Module"，所有业务 Module 都依赖它们：

| Module | 职责 |
|---|---|
| `ModAsset` | YooAsset 封装，统一资源加载入口 |
| `ModAudio` | 音频播放 |
| `ModCamera` | 相机控制 |
| `ModConfig` | 配表加载（Table_*） |
| `ModEvent` | 事件总线（Dispatch / Listen） |
| `ModFsm` | 状态机基类，业务状态机继承它 |
| `ModTimer` | 定时器 |
| `ModUI` | UI 框架（UIView_* 通过它打开） |
| `ModuleBase` | **所有 Module 的基类**——参考 [ModuleBase.cs](../../../Assets/Scripts/Hotfix/Framework/Module/ModuleBase.cs) |

### Common 层（通用业务模块）
`Assets/Scripts/Hotfix/Common/Module/`

跨场景使用的业务模块：

| Module | 职责 |
|---|---|

### Activity 层（活动）
`Assets/Scripts/Hotfix/Activity/`

短期活动模块，命名 `ModActivity_*`：

### InGame 层（游戏内）

## 主工程 ↔ 热更工程的物理隔离

### 主工程**绝不**直接 new 热更工程类
错误：
```csharp
// 在 Assets/Scripts/Main/SomeFile.cs 中
var account = new ModAccount();  // ❌ 编译会失败，且违反隔离原则
```

正确（主工程仅通过委托调热更）：
```csharp
// Boot.cs
private void Update()
{
    Loader.OnUpdate?.Invoke();  // ✅ 热更工程注册委托
}
```

### 热更工程通过 `Loader.OnXxx?.Invoke()` 接收生命周期
热更代码（在 ModFsm / ModuleBase 等）通过 `+=` 订阅这些静态委托即可。

## 资源加载

唯一允许的资源加载方式：
```csharp
Game.GetMod<ModAsset>().LoadAsync<GameObject>(path)  // ✅
```

禁用（除 Boot 阶段外）：
```csharp
Resources.Load<GameObject>(path)                    // ❌
AssetBundle.LoadFromFile(path)                       // ❌
```

Boot 阶段例外见 [Boot.cs:39](../../../Assets/Scripts/Main/Boot.cs#L39) — 启动画面尚未加载 ModAsset 时使用 Resources.Load 是允许的。
