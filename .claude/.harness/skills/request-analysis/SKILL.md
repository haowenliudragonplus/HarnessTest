# Skill: 需求分析（Request Analysis）

> 由 Planner Agent 在阶段 1（需求分析）和阶段 2'（计划返工）启动时加载。

---

## 工作流（按顺序执行）

### Step 1: 理解原始需求
- 把用户原话**完整 quote** 到 spec.md 的"背景"段开头
- 提取三个要素：
  - **WHAT**：做什么（功能 / 修复 / 重构）
  - **WHY**：业务驱动（玩家反馈？运营需求？技术债？）
  - **WHEN**：是否有时间约束（不必猜，没说就不写）

### Step 2: 实测影响面（不要猜）

使用 Grep / Glob 实测**而非主观估计**：

```
- 涉及哪些 Module？→ Grep "class Mod" 找出所有 ModuleBase 派生类，确认本次涉及哪些
- 改动文件清单？→ 根据 WHAT 列出**预计**改动文件，并用 Read 验证文件存在
- 跨链路影响？→ Grep 改动方法名/字段名，看有哪些调用点
- 配表影响？→ 如涉及数值，找 Table_* 中相关项
- Storage 影响？→ 如涉及持久化数据，找 StorageClientCommon 中对应字段
```

将实测结果写入 spec.md 的"影响面"章节。**禁止**写"少量""若干"这种模糊词——给具体数字。

### Step 3: 识别风险

按下表逐项检查：

| 风险类别 | 触发条件 | 应对 |
|---|---|---|
| 主工程改动 | 改了 `Assets/Scripts/Main/` 下文件 | 标"高"，注明需重新打包 App |
| 高敏感模块 | 改了 ModAccount / ModBag / ModIAP / 广告相关 | 标"高"，要求额外冒烟测试 |
| 配表迁移 | 改 Table_* 字段类型或新增必填字段 | 标"中"，需考虑老数据兼容 |
| Storage Schema | 改 StorageClientCommon 字段 | 标"中"，老用户首次登录的迁移逻辑 |
| 网络协议 | 改了 RPC 接口出入参 | 标"高"，服务端配合 + 灰度 |
| FSM 跳转 | 改 ChangeState 调用点 | 标"中"，测试路径多 |
| 仅 UI 改动 | 仅修改 UIView_* prefab/逻辑 | 标"低" |

### Step 4: 列出待决议问题

**绝不**自作主张回答以下类问题，必须在 spec.md 列出来等用户决定：
- "是否需要兼容老版本数据？"
- "新字段配表的默认值是多少？"
- "失败兜底是显示错误还是静默重试？"
- "新功能开关是否走 ABTest？"
- "改动是热更还是必须打包？"

每条问题给出**你的倾向 + 理由**，方便用户快速做决定。

### Step 5: 拆解任务

把 WHAT 拆为可独立完成、可独立验证的 Task。每项的"粒度"控制在：
- 修改/新增 1~3 个文件
- 30 分钟到 2 小时工作量
- 有单一明确的验收点

依赖关系**用 DAG 表示**：标注"Task X 必须在 Task Y 后做"。

### Step 6: 写验收标准（DoD）

可被机器或人工**明确验证**的条件，例如：
- ✅ 好：`在 Unity Editor 中触发 X 操作，UIView_Tip 弹出数量不超过 3`
- ❌ 坏：`效果正常 / 运行流畅 / 用户满意`

## spec.md 模板（必填章节）

参考 [planner.md 第 3 节](../../agents/planner.md#3-specmd-必填章节结构硬约束)，必须包含：

1. 背景
2. 目标
3. 不做什么（Scope Cut）
4. 影响面（实测）
5. 风险
6. 待决议问题
7. 验收标准（DoD）
8. 本轮修订（仅返工版本有此章节）

---

## tasks.md 模板

参考 [planner.md 第 4 节](../../agents/planner.md#4-tasksmd-必填字段)，每项 Task 必含：

- 输入
- 输出
- 验收
- 依赖
- 预估

## 常见错误（评审会被打 MUST FIX）

| 错误 | 为什么不行 |
|---|---|
| 影响面写"少量文件" | 不可验证，禁止模糊词 |
| 待决议被 Planner 自答 | 越权，必须留给用户 |
| Task 验收写"运行正常" | 不可验证 |
| 漏掉对 ModAccount / ModIAP 的影响 | 高敏感模块必须显式标注 |
| 拆分粒度过大（单 Task ≥ 1 天工作量） | 不可独立验证 |
| Task 依赖成环 | 拓扑不可调度 |
