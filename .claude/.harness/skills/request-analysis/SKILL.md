# Skill: 需求分析（Request Analysis）

> 由 Planner Agent 在需求分析阶段与计划返工阶段启动时加载。

---

## 工作流（按顺序执行）

### Step 1: 理解原始需求
- 把用户原话**完整 quote** 到 spec.md 的"背景"段开头
- 提取三个要素：
  - **WHAT**：做什么（功能 / 修复 / 重构）
  - **WHY**：业务驱动（用户反馈？运营需求？技术债？）
  - **WHEN**：是否有时间约束（不必猜，没说就不写）

### Step 2: 实测影响面（不要猜）

使用 Grep / Glob 实测**而非主观估计**。具体抓取方式以本项目代码分层规则（[.claude/.harness/rules/工程结构.md](../../rules/工程结构.md)）为准，至少覆盖：

- 涉及哪些业务模块/分层 → 按项目命名约定 Grep 出对应符号
- 改动文件清单 → 根据 WHAT 列出**预计**改动文件，并用 Read 验证文件存在
- 跨链路影响 → Grep 改动方法名/字段名，看有哪些调用点
- 外部数据影响 → 如涉及配置数据、持久化存储、网络协议，找对应来源
- 部署边界 → 是否涉及主-热更边界、native 模块、CI、资源包等

将实测结果写入 spec.md 的"影响面"章节。**禁止**写"少量""若干"这种模糊词——给具体数字。

### Step 3: 识别风险

按下表逐项检查（具体高敏感模块名以 [.claude/.harness/rules/工程结构.md](../../rules/工程结构.md) 中标注的为准）：

| 风险类别 | 触发条件 | 应对 |
|---|---|---|
| 影响部署边界 | 改动跨越热更/原生/打包边界 | 标"高"，注明需重新部署 |
| 高敏感业务模块 | 改了 rules/工程结构.md 中标记为"高敏感"的模块 | 标"高"，要求额外验证 |
| 配置数据迁移 | 改配置字段类型或新增必填字段 | 标"中"，需考虑老数据兼容 |
| 持久化数据 Schema | 改本地/云端存储字段 | 标"中"，老用户首次进入的迁移逻辑 |
| 网络协议 | 改 RPC 接口出入参 | 标"高"，需服务端配合 + 灰度 |
| 状态机/流程跳转 | 改状态切换点 | 标"中"，测试路径多 |
| 仅 UI / 表现层改动 | 仅修改视图层逻辑 | 标"低" |

### Step 4: 列出待决议问题

**绝不**自作主张回答以下类问题，必须在 spec.md 列出来等用户决定：
- "是否需要兼容老版本数据？"
- "新字段默认值是多少？"
- "失败兜底是显示错误还是静默重试？"
- "新功能是否走灰度 / AB 实验？"
- "改动是局部更新还是需要完整部署？"

每条问题给出**你的倾向 + 理由**，方便用户快速做决定。

### Step 5: 拆解任务

把 WHAT 拆为可独立完成、可独立验证的 Task。每项的"粒度"控制在：
- 修改/新增 1~3 个文件
- 30 分钟到 2 小时工作量
- 有单一明确的验收点

依赖关系**用 DAG 表示**：标注"Task X 必须在 Task Y 后做"。

### Step 6: 写验收标准（DoD）

可被机器或人工**明确验证**的条件，例如：
- ✅ 好：`在本地运行环境触发 X 操作，弹出实例数量不超过 3`
- ❌ 坏：`效果正常 / 运行流畅 / 用户满意`

## spec.md 模板（必填章节）

参考 [planner.md 第 3 节](../../agents/planner.md#3-specmd-必填章节结构硬约束)，必须包含：

1. 背景
2. 目标
3. 不做什么（Scope Cut）
4. 影响面（实测）
5. 风险
6. 待决议问题
7. 验收标准（DoD）
8. 本轮修订（仅返工版本有此章节）

---

## tasks.md 模板

参考 [planner.md 第 4 节](../../agents/planner.md#4-tasksmd-必填字段)，每项 Task 必含：

- 输入
- 输出
- 验收
- 依赖
- 预估

## 常见错误（评审会被打 MUST FIX）

| 错误 | 为什么不行 |
|---|---|
| 影响面写"少量文件" | 不可验证，禁止模糊词 |
| 待决议被 Planner 自答 | 越权，必须留给用户 |
| Task 验收写"运行正常" | 不可验证 |
| 漏掉对 rules/工程结构.md 中标注的高敏感模块的影响 | 高敏感模块必须显式标注 |
| 拆分粒度过大（单 Task ≥ 1 天工作量） | 不可独立验证 |
| Task 依赖成环 | 拓扑不可调度 |
